-T <0-5> Nmap                              скорости  

--min-parallelism <number>                 частоты  

--max-rtt-timeout <time>                   таймаутов тестовых пакетов 

--min-rate <number>                        количества одновременно отправляемых пакетов 

--max-retries <number>                     количества повторных попыток для сканируемых портов, которые должны быть просканированы.





--initial-rtt-timeout слишком короткого начального значения таймаута RTT ( ) может привести к тому, что пропустим некоторые хосты.



--max-retries   способ увеличить скорость сканирования — указать частоту повторных попыток отправки пакетов

&nbsp;Значение по умолчанию — 10, но его можно уменьшить до 0. Это означает, что если Nmap не получит ответа для порта, он больше не будет отправлять пакеты на этот порт и пропустит его





Пример:

sudo nmap 10.129.2.0/24 -F --max-retries 0 | grep "/tcp" | wc -l







--min-rate <number> При установке минимальной скорости для отправки пакетов мы указываем Nmap, что одновременно должно отправляться указанное количество пакетов. Система будет пытаться поддерживать скорость соответствующим образом.



Оптимизированное сканирование

sudo nmap 10.129.2.0/24 -F -oN tnet.minrate300 --min-rate 300
